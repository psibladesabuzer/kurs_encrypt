# Локальное шифрование почтовых вложений

Курсовой проект по теме: "Разработка браузерного расширения для локального шифрования почтовых вложений перед отправкой по электронной почте".

Проект оставлен компактным для защиты:

- `browser_extension` - локальное расширение для Chrome/Edge;
- `kurs_crypto/crypto_service.py` - Python-реализация криптографической схемы;
- `tests` - проверка шифрования, расшифрования и контроля целостности.

## Установка расширения

1. Открыть `chrome://extensions` или `edge://extensions`.
2. Включить режим разработчика.
3. Нажать `Load unpacked` / `Загрузить распакованное расширение`.
4. Выбрать папку `browser_extension`.

## Демонстрация

1. Открыть popup расширения.
2. Выбрать файл-вложение.
3. Ввести пароль и имя результата.
4. Нажать `Выполнить`.
5. Получить файл `.aes256`.
6. Переключить режим на расшифрование и восстановить исходный файл.

Файл не отправляется на сервер: обработка выполняется локально в браузере через Web Crypto API.

## Криптографическая схема

- Алгоритм: AES-256-GCM.
- Генерация ключа: PBKDF2-HMAC-SHA256.
- Длина ключа: 256 бит.
- Соль: 16 байт, генерируется случайно для каждого файла.
- Nonce для GCM: 12 байт, генерируется случайно для каждого файла.
- Число итераций PBKDF2: 300000.
- Контроль целостности: GCM authentication tag, 16 байт.

Формат зашифрованного файла:

```text
MAGIC | VERSION | HEADER_LENGTH | HEADER_JSON | CIPHERTEXT | GCM_TAG
```

В заголовке хранятся открытые параметры, необходимые для расшифрования: алгоритм, KDF, число итераций, salt, nonce, исходное имя файла и время создания. Заголовок передается в AES-GCM как AAD, поэтому изменение параметров обнаруживается при проверке тега.

## Python-проверка алгоритмов

Установка зависимости:

```powershell
py -3.9 -m pip install -r requirements.txt
```

Запуск тестов:

```powershell
py -3.9 -m unittest discover -s tests
```

Python-код нужен для отдельного объяснения алгоритмов шифрования и генерации ключа. Пользовательский функционал демонстрируется через браузерное расширение.
